The Contributor License Agreement assistant (CLA assistant) lets you handle the legal side of contributions to your code easily.

All you need to do is to store your CLA as a GitHub Gist file and link it with your repository in the CLA assistant.

WHAT IT DOES FOR YOU:

  • it notifies each contributor to sign the CLA as soon as he creates a pull request
  • it lets contributors sign your CLA easily
  • it authenticates the signer with his GitHub account
  • it updates the status of the pull request as soon the contributor agrees to your CLA

CLEAR & EASY:

If the content of your CLA changes over time you don't have to notify all of your active contributors to sign a new version of it again.

The CLA assistant will do it for you!

Just update the linked Gist file.

Each new pull request will get a comment with the request to sign the CLA even if the user has already signed the previous version.

You as a repository owner can review the list of users who signed your CLA for each version.

JUST A FEW CLICKS:

2 easy steps to activate your CLA
First: Add your repository from GitHub
Last: Enter the GIST URL to your CLA file
3 easy steps to accept the CLA
First: You create a pull request in GitHub
Second: Accept the CLA
Last: Thank you! You're all set
